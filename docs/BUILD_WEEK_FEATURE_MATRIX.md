# Feature and quirk matrix

This matrix is maintained as a current subsystem inventory. Historical counts
remain in [the Build Week record](BUILD_WEEK.md); future plans do not count as
implemented rows here.

`K`, `M`, and `C` mean computer keyboard, mouse, and configured controller.
The controller action source of truth is `src/navigation.rs`; detailed bindings
are in `CONTROLLER_INTERFACE.md`.

Known product boundaries: Debian/Linux is the supported build family; real
audio requires an already running JACK server; actual MIDI routing requires the
named device; loop playback does not time-stretch; the final bus is limited to
one exact software-monitored stereo input and refuses unconfirmed doubled
direct monitoring; one managed software synth runs at a time; there is no
native sampler, multitrack audio editing timeline, or XM compatibility.
